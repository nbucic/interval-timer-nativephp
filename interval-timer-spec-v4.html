01

Project Overview

▾
Purpose

A demo app showcasing NativePHP Mobile v3.1 — a native, fully offline interval timer running on real Laravel 13 with PHP 8.5, no web server. Users build multi-phase repeatable timers with per-phase pauses and cooldowns, configurable audio countdowns, and a live total duration display.

Key Terminology (canonical)
  • Program — named collection of up to 10 phases + per-program settings
  • Phase — a named timed block with repetitions, a pause between reps, and a cooldown after the final rep
  • Repetition — one execution of the phase duration (max 50 per phase)
  • Pause — dead-time between repetitions within the same phase (not a separate phase)
  • Cooldown — dead-time after the last repetition of a phase, before the next phase begins. Editable on every phase including the last, but the last phase's cooldown is never executed and never counted — a program always ends on the final rep of the final phase
  • Beep Lead-in — per-program (3s or 5s). Seeded from global default on program creation
  • Total Duration — live-computed: all reps + all pauses + cooldowns of non-last phases only. Last phase cooldown excluded always
Phase execution sequence — 3 phases, 2 reps each

Phase 1: [REP 1] → [PAUSE] → [REP 2] → [COOLDOWN]
Phase 2: [REP 1] → [PAUSE] → [REP 2] → [COOLDOWN]
Phase 3: [REP 1] → [PAUSE] → [REP 2] → [COOLDOWN — skipped]
END

Iron rule: a program always ends on the final rep of the final phase. No cooldown, no pause, nothing after it. The last phase's cooldown field is editable (it becomes active if a new phase is added after), but is never executed or counted.

Pause field is hidden when a phase has only 1 repetition — there is nothing to pause between.

Core Functionality

▾
Phases — up to 10, each with up to 50 reps
  • Label — free text ("Sprint", "Rest", "Hold")
  • Duration — seconds per repetition (mm:ss picker)
  • Repetitions — 1–50. At 50, the stepper is disabled
  • Pause — seconds between reps. Hidden when reps = 1 (nothing to pause between). Has beep countdown at the end
  • Cooldown — seconds after final rep. Always visible on every phase. On the last phase: field is editable but greyed with tooltip "not counted — add another phase". Has breathing animation + "Take deep breaths". Has beep countdown at the end. Never executed or counted when it's the last phase.
  • Color — preset palette. Fills the active timer background during reps
  • Drag-to-reorder. At 10 phases the Add button is disabled showing 10/10
Beep / Voice Countdown
  • Global sound mode — Beep (bundled mp3) or Voice (Android TTS, feminine, calm)
  • Per-program lead-in — 3s or 5s. Seeded from global default. User overrides per program
  • Countdown fires at end of: every rep, every pause, every cooldown
  • If phase/pause/cooldown duration < lead-in → countdown starts from second 1
  • Voice says: "five… four… three… two… one" (or from three). Android TTS API, no pre-recorded files
End-of-Program Sound
  • Triple beep — three rapid beeps 150ms apart (finish-triple.mp3)
  • Chime — tonal completion sound (finish-chime.mp3)
  • Chosen per-program, seeded from global default
Cooldown — Breathing Animation
  • Cooldown state shows a large timer with a slow pulsing glow (8s cycle: 4s inhale, 4s exhale)
  • "Take deep breaths" message displayed beneath the timer
  • Phase background switches to a neutral dark tone during cooldown
  • Future hook: WearOS heart rate data drives animation cadence in Phase 4
App Lifecycle Behaviour
  • Phone call received — auto-pause (PhoneStateListener). Auto-resume when call ends. Single "paused" beep fires
  • App backgrounded — timer keeps running via Android Foreground Service. Notification shows phase + time
  • App killed — state discarded silently. No history entry written. History only records fully completed programs
Total Duration — always visible

Formula: for every phase except the last: (duration × reps) + (pause × (reps−1)) + cooldown. For the last phase: (duration × reps) + (pause × (reps−1)) — cooldown excluded. Shown live in the editor, on each library card, and as a secondary "total remaining" countdown in the active timer.

Architecture

▾
TimerRunner State Machine
idle → running // executing a repetition → paused // user hit pause (or phone call) → pause // dead-time between reps → cooldown // dead-time after final rep, breathing animation → completed // all phases done, end sound fires
Execution Cursor (PHP 8.5 readonly class)
readonly class TimerCursor { public function __construct( public int $phaseIndex, public int $repIndex, public string $state, // running|pause|cooldown|paused public int $remaining, // seconds left in current segment public int $totalRemaining ) {} // PHP 8.5 clone with — immutable advance public function nextRep(): self { return clone($this, ['repIndex' => $this->repIndex + 1, 'state' => 'running']); } }
PHP 8.5 Pipe operator in TimerProgram
public static function load(string $id): self { return Storage::get("programs/{$id}.json") |> json_decode(...) |> fn($d) => self::validate($d) |> fn($d) => self::hydrate($d); }
